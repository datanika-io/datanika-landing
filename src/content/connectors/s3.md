Amazon S3 is the most common file-based source — teams use it to ingest CSV exports, Parquet data lake files, JSON event logs, and third-party data drops into their warehouse. Datanika reads files from S3 using the dlt filesystem source, which handles pagination, glob patterns, and schema inference automatically.

> **S3 is source-only.** You can read files from S3 but can't use S3 as a destination in Datanika.

## Prerequisites

- A **Datanika account** with permission to create connections.
- A **destination warehouse** already connected.
- An **AWS account** with an S3 bucket containing the files you want to sync.
- **IAM credentials** with `s3:GetObject` and `s3:ListBucket` permissions on the target bucket.

## Step 1 — Create IAM credentials

1. In the AWS Console, go to **IAM → Users → Create user**.
2. Name it `datanika-s3-reader`.
3. Attach a policy with minimum permissions:
   ```json
   {
     "Version": "2012-10-17",
     "Statement": [{
       "Effect": "Allow",
       "Action": ["s3:GetObject", "s3:ListBucket"],
       "Resource": [
         "arn:aws:s3:::<your-bucket>",
         "arn:aws:s3:::<your-bucket>/*"
       ]
     }]
   }
   ```
4. Create an access key: **Security credentials → Create access key → Third-party service**.
5. Copy the **Access Key ID** and **Secret Access Key**.

> **Least privilege.** `GetObject` + `ListBucket` is all Datanika needs. No write, delete, or bucket management permissions.

![Creating the IAM user](/docs/connectors/s3/01-credentials.png)

## Step 2 — Add the connection in Datanika

1. In Datanika, open **`/connections`** and pick **S3** from the type dropdown at the top of the inline New Connection form.
2. Fill in:
   - **Bucket URL** — `s3://<your-bucket>` or `s3://<your-bucket>/path/prefix/`
   - **AWS Access Key ID** — from Step 1 (optional if running on EC2 with an IAM role)
   - **AWS Secret Access Key** — from Step 1 (optional with IAM role)
   - **Region** — e.g. `us-east-1` (optional, auto-detected from bucket)
   - **Endpoint URL** — only needed for S3-compatible stores (MinIO, Backblaze B2, etc.)
3. Click **Create Connection**.

![Adding S3 in Datanika](/docs/connectors/s3/02-add-connection.png)

## Step 3 — Configure files and schemas

1. Open the connection and click **Configure pipeline**.
2. Pick the destination and target schema (e.g. `raw_s3`).
3. Set the **file glob** pattern to match the files you want:
   - `*.csv` — all CSV files in the bucket prefix
   - `events/*.parquet` — Parquet files in the events/ subfolder
   - `**/*.json` — all JSON files recursively
4. Datanika infers the schema from the first batch of files. Use `replace` for full reloads or `append` for new-files-only.
5. Save.

## Step 4 — First run

1. Click **Run now**. dlt streams files from S3 and auto-parses CSV/JSON/Parquet.
2. Browse **Catalog → `raw_s3`** to verify tables. Each file type typically maps to one table.

![First run](/docs/connectors/s3/04-first-run.png)

## Step 5 — Schedule it

Daily or every 6 hours is typical for file-based sources. Match the cadence to how often new files land in S3.

## Troubleshooting

### `Access Denied`
**Fix.** Verify the IAM policy has `s3:GetObject` + `s3:ListBucket` on both the bucket ARN and `bucket/*`.

### No files found
**Fix.** Check the bucket URL prefix and file glob pattern. The glob is relative to the bucket URL — if your bucket URL is `s3://my-bucket/data/` and your glob is `*.csv`, it looks for `s3://my-bucket/data/*.csv`.

### Schema inference picks wrong types
**Fix.** For CSV files without headers, or with inconsistent types, dlt may infer incorrect column types. Check the first few files for consistency.
